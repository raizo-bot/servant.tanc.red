Commands

  • Giveaway

    Description

    Alias

    Start a giveaway that draws a given amount of people as winners after a given time.
    List all running giveaways of the current server.

    None

    Permissions

    User Servant
    None Read Text Channels & See Voice Channels, Send Messages, Read Message History, Embed Links, Add Reactions

    Usage

    Start giveaway
    Command: !giveaway "[prize name]" [amount of winners] [time]
    Example: !giveaway "100 Cookies" 1 12h

    List giveaways
    Command: !giveaway list

    Hint

    "Time formats: d = days, h = hours, m = minutes

    • Also, you can use as many time-arguments as you want
    • Example: giveaway "100 cookies" 1 2d 12h 36m 1d
    • Time result: 2d + 12h + 36m + 1d = 3 days, 12 hours, 36 minutes
    Seconds are not supported.

  • Poll

  • RemindMe

    Description

    Alias

    Set up an RemindMe to a specific time.
    Other members can upvote the RemindMe to also get reminded.
    The RemindMe will have a jump URL to the context.

    reminder
    alarm

    Permissions

    User Servant
    None Read Text Channels & See Voice Channels, Send Messages, Read Message History, Embed Links

    Usage

    Command: !remindme "Optional Topic" [time]

    Example 1 → Using Time Arguments
    With topic: !remindme "Pat Servant" 3d 2h 1m
    Without topic: !remindme 3d 2h 1m

    Example 2 → Using a Date and Time
    With topic: !remindme "Pat Servant" 2020-31-12 23:59
    Without topic: !remindme 2020-31-12 23:59

    Hint

    - The command author will always be reminded, even without upvoting.
    - A RemindMe must not be further in the future than 1 month.

    Time Arguments
    - Time formats: d = days, h = hours, m = minutes
    ⤷ Also, you can use as many time arguments as you want
    ⤷ Example: !remindme "Pat Servant" 2d 12h 36m 1d
    ⤷ Time result: 2d + 12h + 36m + 1d = 3 days, 12 hours, 36 minutes
    - Seconds are not supported.

    Date and Time
    - You cannot set reminders in the past.
    - You have to use the 24-hour system.
    - It takes your timezone into account.
    ⤷ If you didn't set one up, it will use UTC.
    ⤷ To set one up, use the !user command.

  • Signup

    Description

    Alias

    Let people sign up for an event (e.g. a raid).
    The signup will be completed if either of the following happens:
    1. The given amount of people have signed up (reacted).
    2. The given expiring date (or the default one if not given) was reached (see last hint).
    3. The signup creator clicked the :x: reaction.

    event

    Permissions

    User Servant
    None Read Text Channels & See Voice Channels, Send Messages, Read Message History, Embed Links, Add Reactions

    Usage

    Command: !signup "Title" [amount of participants] [event date and time]
    Example 1: !signup "Holy Grail War" 10 2019-12-31 23:30
    Example 2: !signup 10 2019-12-31 23:30
    Example 3: !signup "Holy Grail War" 10
    Example 4: !signup 10

    Hint

    • The amount of participants has to be within 1 and 100.
    • The event date and time has to be within the next 4 weeks.
    • If you don't set the event date and time, the signup will expire in 4 weeks.
    • The event date and time will use the server's offset. You can check it by typing !serverinfo -> Servant Settings -> Offset
    • If you set an event date, the signup will expire 30 minutes earlier, so you have time to organize the group.

  • Timezone

    Description

    Alias

    Convert a date and time from one timezone to another.

    None

    Permissions

    User Servant
    None Read Text Channels & See Voice Channels, Send Messages, Read Message History, Embed Links

    Usage

    Command: !timezone yyyy-MM-dd HH:mm [current timezone] [target timezone] Example: !timezone 2019-01-01 22:00 PST CET

    Hint

    This command uses the 24 hour system.
